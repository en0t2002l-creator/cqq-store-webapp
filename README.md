# CQQ SHOP - Интернет-магазин винтажной одежды

Стильный одностраничный сайт для продажи б/у одежды с админ-панелью и интеграцией с Telegram.

## 🎨 Особенности

- **Элегантный дизайн** в стиле модельного агентства
- **Галерея товаров** с предпросмотром до 3 фото при наведении
- **Детальные карточки товаров** с возможностью просмотра до 10 фотографий
- **Админ-панель** для управления товарами (добавление, редактирование, удаление)
- **Управление статусом** товаров (в продаже/продано)
- **Размеры и количество** в наличии для каждого товара
- **Интеграция с Telegram** для связи с покупателями
- **Адаптивный дизайн** для всех устройств
- **Локальное хранилище** данных (LocalStorage)

## 🚀 Развертывание на GitHub Pages

### Шаг 1: Создание репозитория

1. Войдите в свой аккаунт GitHub
2. Создайте новый репозиторий (например, `cqq-shop`)
3. Сделайте репозиторий публичным

### Шаг 2: Загрузка файлов

1. Скачайте все файлы проекта:
   - `index.html`
   - `script.js`
   - `README.md`

2. Загрузите их в репозиторий через веб-интерфейс GitHub или через git:

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ваш-username/cqq-shop.git
git push -u origin main
```

### Шаг 3: Активация GitHub Pages

1. Перейдите в Settings репозитория
2. В меню слева выберите "Pages"
3. В разделе "Source" выберите:
   - Branch: `main`
   - Folder: `/ (root)`
4. Нажмите "Save"

Через несколько минут сайт будет доступен по адресу:
`https://ваш-username.github.io/cqq-shop/`

## ⚠️ Решение проблем с GitHub

### Ошибка 500/503 при git push

Если при загрузке файлов вы получаете ошибки:
```
unable to access 'https://github.com/...': The requested URL returned error: 500
unable to access 'https://github.com/...': The requested URL returned error: 503
```

**Причины и решения:**

1. **Временные проблемы с серверами GitHub**
   - Подождите 5-10 минут и попробуйте снова
   - Проверьте статус GitHub: https://www.githubstatus.com/

2. **Проблемы с размером файлов**
   - Убедитесь, что файлы не слишком большие
   - Наш проект очень маленький, это не должно быть проблемой

3. **Используйте веб-интерфейс GitHub (самый простой способ)**
   - Вместо git команд, загрузите файлы через браузер:
   - Откройте ваш репозиторий на GitHub.com
   - Нажмите "Add file" → "Upload files"
   - Перетащите все 3 файла (`index.html`, `script.js`, `README.md`)
   - Нажмите "Commit changes"

4. **Альтернатива: GitHub Desktop**
   - Скачайте GitHub Desktop: https://desktop.github.com/
   - Клонируйте репозиторий через приложение
   - Скопируйте файлы в папку репозитория
   - Сделайте commit и push через интерфейс приложения

5. **Проверьте интернет-соединение**
   - Убедитесь, что ваше соединение стабильно
   - Попробуйте использовать VPN если есть проблемы с доступом

### Ошибка "The process '/usr/bin/git' failed with exit code 128"

Это обычно означает проблему с GitHub Actions. Решение:

1. Зайдите в Settings репозитория
2. Перейдите в Actions → General
3. В разделе "Workflow permissions" выберите "Read and write permissions"
4. Нажмите "Save"
5. Попробуйте снова

### Рекомендуемый метод загрузки (самый надежный)

**Через веб-интерфейс GitHub:**

1. Создайте новый репозиторий на GitHub.com
2. Не добавляйте README, .gitignore или лицензию (создайте пустой репозиторий)
3. После создания нажмите "uploading an existing file"
4. Перетащите все 3 файла: `index.html`, `script.js`, `README.md`
5. Нажмите "Commit changes"
6. Перейдите в Settings → Pages и активируйте GitHub Pages
7. Готово! ✅

## 🔧 Настройка

### Изменение пароля админ-панели

**Текущий пароль:** `cqqshop200226`

Пароль хранится в виде SHA-256 хеша для безопасности. Чтобы установить свой пароль:

1. Откройте сайт в браузере
2. Нажмите F12 (открыть консоль разработчика)
3. Во вкладке Console введите команду:
```javascript
await hashPassword('ваш_новый_пароль')
```
4. Скопируйте полученный хеш (длинная строка из букв и цифр)
5. Откройте файл `script.js`
6. Найдите строку с `PASSWORD_HASH` и замените значение на ваш хеш:

```javascript
const PASSWORD_HASH = 'ваш_скопированный_хеш';
```

**Почему это безопаснее?**
- Пароль не хранится в открытом виде в коде
- Даже если кто-то увидит хеш на GitHub, он не сможет узнать ваш пароль
- Хеш невозможно расшифровать обратно в пароль

### Изменение Telegram username

В файле `script.js` найдите и измените:

```javascript
const TELEGRAM_USERNAME = '@vizkazz'; // Ваш username
```

## 📊 Хранение данных

Все данные хранятся в LocalStorage браузера. Это означает:

- ✅ Данные сохраняются между сеансами
- ✅ Не требуется сервер или база данных
- ⚠️ Данные хранятся только в браузере (при очистке кэша будут удалены)
- ⚠️ Разные браузеры = разные данные

**Рекомендация:** Периодически делайте резервную копию данных через консоль браузера:

```javascript
// Экспорт данных
console.log(localStorage.getItem('cqqshop_items'));

// Импорт данных
localStorage.setItem('cqqshop_items', 'здесь_ваши_данные');
```

## 🔒 Безопасность

✅ **Улучшенная безопасность:** Пароль хранится в виде SHA-256 хеша, а не в открытом виде.

**Как это работает:**
- Когда вы вводите пароль, он хешируется и сравнивается с сохраненным хешем
- Даже если кто-то увидит код на GitHub, он не сможет узнать ваш пароль
- Хеш невозможно расшифровать обратно в исходный пароль

**Важно знать:**
- Регулярно создавайте резервные копии данных
- Не используйте слишком простые пароли типа '123456'
- При смене пароля сразу загружайте обновленный файл на GitHub

Для максимальной защиты коммерческих данных рекомендуется серверное решение с настоящей базой данных.
